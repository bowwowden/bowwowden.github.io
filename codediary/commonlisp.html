Common Lisp

This is an overview of the Common Lisp programming language.

Common Lisp is a functional language much like Scheme. It is often grouped in with functional languages. However, it is not a pure functional language like Ocaml, Pure, or F#. Since it allows functions to not have exactly one argument, it is not strict to the lambda calculus. It is a language that goes back to the 1950's, back when computers were only used on mainframes in universities or military.

Benefits of Functional over Imperative languages:

Functional programming is a liberation from von Neumann-style imperative programming. It has benefits in parallel processing, redefining language semantics on the fly, and avoiding side-effects from using variables in stored memory.
